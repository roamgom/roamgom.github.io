Decorator

파이썬으로 어느정도 작업하다보면 코드 중에 아래와 같은 형태로 된 코드를 볼 수 있다.
(아래는 Flask 예제)

위의 @app.route()와 같은 형태를 Decorator라고 부른다.
Decorator를 사용하는 이유는 대상 함수를 다른 함수로 wrapping 하여 대상함수의 실행 전후에 추가적으로 실행을 하고 싶은 기능을 재사용 가능한 형태로 구현해준다.

예를 들어, 웹 서비스를 만든다고 했을 때, 각 API마다 로그를 남기고 싶다고 하자.
필요한 기능을 로그인, 글쓰기, 사진올리기 총 3개의 함수로 나누어 생각할 때 각 함수마다 로그를 남기는 코드를 적용해줘야 한다.

보통은 위와같이 로그를 남기는 함수를 선언하여 각 메소드 안에서 호출하겠지만 Decorator를 사용하면 아래와 같은 형태로도 가능하다.

Decorator를 사용할 경우 여러번 재사용 함으로써, 함수에 대한 가독성이 훨씬 좋아진 코드를 볼 수 있다. 또한, 같은 패턴을 사용해도 훨씬 더 간편한 코드 작성이 가능하다.

Decorator에 대해 설명을 하자면,
Decorator 함수를 먼저 정의한 후, decorator를 적용시킬 함수를 인자로 받는다. decorator 함수 내부에는 nested 함수를 선언하여 여기에 추가 작업을 선언하여 사용하고 마지막으로 decorator 함수에서 해당 nested 함수를 반환해주면 된다.

func() : 처리 함수
pre_handle() : 추가 처리 코드
decoration_function() : Decorator 함수 (호출용)
wrapper_function() : 처리 함수를 감싸서 추가 처리를 하는 함수 (추가 처리)

위와 같이 이해하면 된다.

Python Generator

Posted on 2020-03-03 Edited on 2020-03-10

Generator

Generator는 Iterator의 심화버전이라 생각할 수 있다. 일반 Iterator의 경우 전제 배열을 반환하지만, Generator는 한번에 하나의 값만 반환하기 때문에 적은 메모리를 사용한다.

Generator 함수를 실행하면, 처음으로 만나는 yield 에서 값을 반환한다.
만약, 다시 한번 함수가 호출되면 그 다음 부분부터 yield 를 만날때까지 코드가 실행된다.

지역변수들은 iteration이 진행될 동안 유지된다. 더 이상 yield 할 값이 없을 경우, StopIteration exception이 발생한다.

Generator 함수를 변수에 할당하게 되면 해당 변수는 Generator 클래스 객체가 된다. 위의 경우, g = gen()에서 g는 Generator 클래스 객체이며, next() 내장함수를 통해 Generator의 다음 값을 불러올 수 있는 것이다.

Iterator와 Generator의 차이

Iterator의 경우, 모든 값이 다 들어간 상태에서 객체를 반환하며, Generator는 yield에 의해 하나의 데이터씩 반환하는 차이다.
Generator는 데이터가 많아 한번에 모든 데이터를 반환하기 어려운 상황에서 일부 데이터만 처리할 때 유용하다.
또한 Iterator를 사용했을 때보다 Generator를 사용하면 더 빠른 시간에 적은 코드와 메모리로 실행이 가능하다. 왜 이게 가능할까?
